Задача 2

  2.1 Высоконагруженная база данных MySql, критичная к отказу
Разместил бы на физическом сервере для большей производительности и доступа к железу напрямую, вопрос отказоустойчивости можно решить обьеденив несколько физ. серверов в кластер средствами бд.
  2.2 Различные web-приложения
Их было бы правильние разместить на виртуализации уровня ОС, удобнее масштабировать, нет потребности в прямом доступе к железу и легче организовывать сложные системы внутри одного серверного пространства(разместить много контейнеров или поов и настроить для них сеть и тд)
  2.3 Windows-системы для использования бухгалтерским отделом
В целом достаточно паравиртуализации(одной windows server с настроенным терминальным сервером) для пользователей, если хотим хостить 1с и бд для нее то правильно будет выделить физический сервер(если верить статьям Гилева так 1с работает быстрее всего), но и на ВМ она тоже нормально себя чувствует.
  2.4 Системы, выполняющие высокопроизводительные расчёты на GPU
Их бы по аналогии с БД вынес на отдельный физический сервер для большей производительности, но есть и вариант с контейнеры Docker с nvidia-container-toolkit
